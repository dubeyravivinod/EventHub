# EventHub

EventHub is a small Django project for managing events and attendee reservations. It includes three primary apps:

- `event` — event models, views and templates for listing, creating, updating and deleting events.
- `reservation` — reservation models and views for attendees to reserve event seats.
- `user` — simple user profile, registration and authentication helpers.

This README covers how to set up and run the project locally, describes important files, and includes troubleshooting notes.

Prerequisites
-------------
- Python 3.8+ (project was developed with Python 3.10+)
- pip
- (optional) a virtual environment

Quick start (development)
-------------------------
1. Create and activate a virtual environment:

```bash
python -m venv .venv
# Windows
.venv\Scripts\activate
# macOS / Linux
source .venv/bin/activate
```

2. Install Django (and any requirements you need). There is no requirements.txt in the repo; install Django compatible with this project (the project header shows Django 6.0.5 was used):

```bash
pip install Django==6.0.5
```

3. Create and apply migrations (important — the reservation model recently added a `user` field):

```bash
# from repo root where manage.py is located
python manage.py makemigrations
python manage.py migrate
```

4. (Optional) Create a superuser to access Django admin:

```bash
python manage.py createsuperuser
```

5. Run the development server:

```bash
python manage.py runserver
```

6. Open http://127.0.0.1:8000/events/ (events list) and http://127.0.0.1:8000/reservations/ to test the reservation views.

Authentication and behavior notes
---------------------------------
- Registration and login views live in the `user` app and have templates at [user/templates/user/register.html](user/templates/user/register.html) and [user/templates/user/login.html](user/templates/user/login.html).
- Reservation creation requires a logged-in user; the code sets the reservation's `attendee_name` and `attendee_email` from the authenticated user's account. The reservation create view uses `LoginRequiredMixin`.
- Reservation list is filtered to show only the logged-in user's reservations. Update and delete views are restricted to the reservation owner.
- The events list template disables the Create Event action for anonymous users and shows login/register links.

Media / static
--------------
- A `profile_pictures/` folder exists in the project root for uploaded user profile images. The project does not yet configure `MEDIA_URL` / `MEDIA_ROOT` in settings.py — add them if you plan to serve uploads in development.

Troubleshooting
---------------
- OperationalError: "no such column: reservation_reservation.user_id"
  - This means the database schema is out-of-date. From the project root run:

```bash
python manage.py makemigrations reservation
python manage.py migrate
```

  - Then restart the dev server.

- If Django can't be imported when running manage.py, ensure your virtualenv is activated and Django is installed.

Development notes & suggestions
------------------------------
- The project uses class-based views for CRUD operations and function views for some user actions.
- Consider adding `MEDIA_URL` and `MEDIA_ROOT` to settings and serving media files in development by adding the proper `urlpatterns` in `eventhub/urls.py`.
